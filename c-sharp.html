What are the main concepts of OOPS? What are Classess and Objects in C#?

Main concepts of OOPS are Objects, Classes, Inheritance, Polymorphism, Abstraction and Encapsulation. A class is a LOGICAL UNIT or BLUEPRINT that contains fields, methods and properties. Class members are:
1. Constructor is a method in the class which gets executed when a class object is created.
2. A field is a variable of any type. It is basically the data.
3. A property is a member that provides helps in read and write of private field.
4. A method is a code block that contains a series of statements.

Object - An object is an INSTANCE of a class.

What is Inheritance? Why Inheritance is important?

Inheritance is creating a PARENT-CHILD relationship between two classes where child class will automatically get the properties and methods of the parent.
Inheritance is good for: REUSABILITY and ABSTRACTION of code.

How to prevent a class from being Inherited?

By using SEALED keyword in class

What is Abstraction?

Abstraction means showing only required things and hide the BACKGROUND details.

What is Polymorphism and what are its types?

Polymorphism is the ability of a variable, object, or function to take on MULTIPLE FORMS.

What is Encapsulation?

Encapsulation means WRAPPING of data and methods/properties into a single unit.

What is Method Overloading? In how many ways a method can be overloaded?

Method Overloading is a type of polymorphism in which we can create multiple methods of the same name in the same class, and all methods work in different ways.
It can be done in 3 ways:
Number of parameters are different
Type of parameters are different
Order of parameters are different

What is the difference between Overloading and Overriding?

1. Method Overloading is a type of polymorphism in which we can create multiple methods of the same name in the same class, and all methods work in different ways.

1. Method Overriding is having methods with the SAME NAME and SIGNATURE but in different classes.

2. Method Overloading will not use any such kewords.

2. Overriding uses VIRTUAL keyword for base class method and OVERRIDE keyword for derived class method.

3. Method overloading doesn't need inheritance. It is in same class.

3. Method overriding NEEDS INHERITANCE(Base class/Derived class). It is not possible in same class.

What is the difference between Method Overriding and Method Hiding?

In Method Hiding, you can hide the implementation of the methods of a base class from the derived class using the new keyword

What are the advantages and limitations of OOPS?

Advantages of OPPS:
1. Reuse of code through inheritance
2. Flexibility through polymorphism
3. Security of data through Data hiding/Encapsulation
4. Modularity for easier troubleshooting
Disadvantages of OOPS:
It is not suitable for small applications

What is the difference between an Abstract class and an Interface (atleast 4)?

1. Abstract class contains both DECLARATION & DEFINATION of methods.
1. Interface contains ONLY DECLARATION of methods.
2. Abstract class contains ABSTRACT keyword.
2. Interface contains INTERFACE keyword.
3. Abstract class can contain methods, fields, constructor and other class members.
3. Inter can contain method only nothing else
4. Abstract class does not support MULTIPLE INHERITANCE.
4. Interface support multiple inheritance.

When to use Interface and when Abstract class?

Abstract class is a good choice when we are sure some method are concrete/defined and must be implemented in the SAME WAY in all derived classes.
An Interface is a good choice when we know a method has to be there, but it can be implemented DIFFERENTLY by independent derived classes.

Why to even create Interfaces?

By using interface we are making sure that. An interface which contains definitions for a group of methods that a non-abstract class or a struct must implement.
By using interfaces, we can also, include behavior of multiple inheritance of classes.

Do Interface can have a Constructor?

NO. Interface can only be derived from.

Can you create an instance of an Abstract class or an Interface?

NO. Abstract class can only be used for inheritance not for object creation.

What are Access Specifiers? What is the default access modifier in a class?

Access specifiers are keywords to specify the accessibility of a class, method, property, field.
The keywords are - Public, Private, Protected, Internal, ProtectedInternal
Note: Internal is the default access modifier of a class

What is Boxing and Unboxing?

The process of converting from a value type to a reference type is called boxing.
And converting reference type to value type is called unboxing
